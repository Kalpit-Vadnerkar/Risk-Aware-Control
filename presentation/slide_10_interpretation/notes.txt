The "hold and confirm" logic is the key design insight from the empirical results. Dr. Pisu will recognize this as a two-stage detection pattern common in fault detection and isolation (FDI) systems: a fast first-stage test (topology + kinematics) gates a slower but more sensitive second-stage test (residual). Only if the first stage passes do we run the more expensive (or more uncertain) second stage. This framing maps directly onto his controls background.

The identifiability result is a theoretical contribution in itself — it formally proves that no trajectory-based signal can close this particular prediction gap. This is worth presenting clearly: we ran a systematic empirical test of seven different signals (max NLL, CVaR, obj_dist_residual, obj_dist_slope, end-of-run distance, fraction of close-distance windows, has_adjacent_lane) and showed that NONE of them discriminate during the approach phase. This exhaustive negative result is publishable on its own (it closes off a direction before others go down it) and motivates the alternative framework design.

Be prepared for the question: "Can the model be retrained to predict this?" Answer: No — not with trajectory data alone. The training data would need to label "this deceleration is toward an obstacle" vs. "this deceleration is toward a signal," and that label is not derivable from the trajectory itself. You would need semantic scene understanding (obstacle classification + driveable path check), which is exactly what Signals 1 and 2 provide.
